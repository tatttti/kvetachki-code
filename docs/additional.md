# Дополнительная спецификация

Дополнительная спецификация определяет нефункциональные требования к системе Flower Greenhouse System.

---

## 1. Нефункциональные требования

### 1.1 Надёжность
- Корректная обработка ошибок.
- Атомарность операций записи.
- Проверка корректности входных данных.

### 1.2 Удобство использования
- Интуитивный консольный интерфейс.
- Минимальное количество шагов для выполнения операций.

### 1.3 Производительность
- Время отклика не более 0.5 секунды.
- Быстрый запуск приложения.

### 1.4 Сопровождаемость
- Модульная структура проекта.
- Организация кода по архитектурным слоям.
- Makefile для сборки.

### 1.5 Безопасность
- Ограничение доступа к административным функциям.
- Проверка корректности входных данных.

### 1.6 Тестирование и покрытие тестами
- Наличие модульных тестов для ключевых модулей.
- Проверка CRUD‑операций, расчёта стоимости, обработки ошибок.
- Автоматический запуск тестов в CI/CD.
- Тесты должны быть независимыми и повторяемыми.

---

## 2. Общие функциональные требования

- Логирование ошибок.
- Проверка корректности данных.
- Автоматическое создание базы данных.
- Обработка исключительных ситуаций.
- Поддержка целостности данных.

---

## 3. Ограничения

- Консольный интерфейс.
- SQLite в качестве СУБД.
- Кроссплатформенность.
