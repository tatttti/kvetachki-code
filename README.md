# Flower Greenhouse System

## Описание проекта

Консольное приложение на языке **C** для управления цветочной оранжереей. Система позволяет управлять каталогом цветов и композиций, оформлять заказы с автоматическим расчётом стоимости, применять надбавки за срочность, генерировать аналитические отчёты и разграничивать доступ (администратор / покупатель).

### Ключевые возможности

- Управление цветами и композициями (CRUD)
- Оформление заказов с расчётом стоимости
- Надбавки за срочность (1день +25%, 2дня +15%)
- Ограничение повышения цены цветка (не более 10%)
- Аналитические отчёты (выручка, популярные композиции, статистика)
- Аутентификация и разграничение прав (ADMIN / CUSTOMER)
- Транзакционность (BEGIN / COMMIT / ROLLBACK)

## Установка и запуск

### Требования

```bash
# Arch Linux / EndeavourOS
sudo pacman -S gcc sqlite3 make python-pip
pip install gcovr

# Ubuntu / Debian
sudo apt install gcc sqlite3 libsqlite3-dev make gcovr
```

### Сборка проекта

```bash
make
```

### Запуск программы

```bash
make run
```

### Очистка

```bash
make clean        # удаление объектных файлов и исполняемого
make clean-all    # полная очистка (удаление build/ и bin/)
```

## Бизнес-правила

### Надбавка за срочность

| Срок выполнения | Надбавка |
|-----------------|----------|
| 1 день | **+25%** |
| 2 дня | **+15%** |
| Обычный | **0%** |

### Ограничение повышения цены

> Цена цветка не может быть повышена, если это увеличит стоимость **любой** композиции более чем на **10%**.

### Транзакционность

При создании заказа используется транзакция:
- `BEGIN TRANSACTION` – начало
- `COMMIT` – фиксация при успехе
- `ROLLBACK` – откат при ошибке

## GitHub Actions

При каждом пуше в ветки `main` или `development` автоматически выполняется:

```yaml

Install dependencies (SQLite3, gcc, gcovr)
Build project (make)
Run unit tests (make test-coverage)
Generate coverage report
Upload coverage report as artifact
```
